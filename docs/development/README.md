## 使用方法

### 1. 准备阶段

确保已满足前置条件：
- Node.js >= 18.17.0
- pnpm >= 8.0.0
- GitHub、Vercel、Supabase 账号

### 2. 按步骤执行

从 [开发步骤.md](./开发步骤.md) 中的 Step 1 开始，按顺序执行每个步骤。

每个步骤包含：
- **目标** - 要完成的任务
- **前置依赖** - 需要先完成哪些步骤
- **执行提示词** - 直接复制发送给 Claude 的指令（含自动验证）
- **参考文档** - 相关设计/需求文档链接

### 3. 示例工作流

```bash
# 1. 阅读当前步骤的说明
# 例如：Step 1 - 创建 Next.js 项目

# 2. 复制"执行提示词"发送给 Claude

# 3. Claude 会自动执行命令、创建文件

# 4. Claude 会自动运行验证检查
#    - 文件是否存在
#    - TypeScript 类型检查
#    - 构建是否成功

# 5. 查看验证结果，通过后继续下一步
```

### 自动验证说明

每个步骤的执行提示词末尾都包含 **【自动验证】** 指令，AI 完成任务后会自动执行以下验证：

| 验证类型 | 命令/方法 | 说明 |
|---------|----------|------|
| 文件检查 | Glob/Read | 验证必需文件/目录是否已创建 |
| 类型检查 | `npx tsc --noEmit` | 检查 TypeScript 类型错误 |
| 构建检查 | `pnpm build` | 检查项目是否能成功构建 |
| 逻辑验证 | 代码审查 | 验证代码逻辑是否完整 |

如验证失败，AI 会自动修复问题并重新验证。

## 常见问题

### Q: 可以跳过某些步骤吗？

A: 不建议。每个步骤都有前置依赖，跳过可能导致后续步骤无法正常执行。

### Q: 某个步骤失败了怎么办？

A: 根据错误信息排查，常见原因：
- 环境变量未配置
- 依赖安装失败
- Supabase 连接问题

参考 [开发步骤.md](./开发步骤.md) 附录中的故障排查表格。

### Q: 可以修改执行提示词吗？

A: 可以，但建议保持提示词的核心指令不变，仅添加额外的定制需求。
